    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a name="user-content-viewblurrer-for-android" class="anchor" href="#viewblurrer-for-android"><span class="octicon octicon-link"></span></a>ViewBlurrer (for Android)</h1>

<p>Little nifty helper to apply a blur-effect to any View. Its convenience lies in it acting as a drop-in replacement for the View that should be blurred, which means that it does not require any changes to your XML layouts. </p>

<h1>
<a name="user-content-how-it-works" class="anchor" href="#how-it-works"><span class="octicon octicon-link"></span></a>How it works</h1>

<p>When creating a <code>ViewBlurrer</code> instance, you pass it the <code>View</code> that you want it to blur.
ViewBlurrer creates a new <code>FrameLayout</code> and adds the passed <code>View</code> and an <code>ImageView</code> to it, which overlays your <code>View</code> element perfectly and be used to hold the blurred version of it.</p>

<h1>
<a name="user-content-setup" class="anchor" href="#setup"><span class="octicon octicon-link"></span></a>Setup</h1>

<p>Note: this is the setup routine for Eclipse/ANT.</p>

<ol>
<li>Import <code>ViewBlurrer</code> as a new Android Project or simply download <a href="https://github.com/orhnsnmz/ViewBlurrer/blob/master/ViewBlurrer/bin/viewblurrer.jar">the .jar file</a> and add it to your build path.</li>
<li>Add the following lines to your project.properties file (this references the <a href="http://android-developers.blogspot.de/2013/09/renderscript-in-android-support-library.html">RenderScript support library</a>):</li>
</ol><pre><code>renderscript.target=19
enderscript.support.mode=true
sdk.buildtools=19.0.1* 

* Replace this with the most recent version of the build tools
</code></pre>

<h1>
<a name="user-content-usage" class="anchor" href="#usage"><span class="octicon octicon-link"></span></a>Usage</h1>

<ol>
<li><p>Crate a new instance of <code>ViewBlurrer</code> and pass it the <code>View</code> element you want to have blurred</p></li>
<li>
<p>Optional: Setup the <code>ViewBlurrer</code> by setting its options:</p>

<ul>
<li>Set the sample size (similar to <a href="http://developer.android.com/reference/android/graphics/BitmapFactory.Options.html#inSampleSize">BitmapFactory.inSampleSize</a>) by calling <code>setSampleSize(x)</code>, where x is an <code>int</code> value &gt;= 1. Higher values improve the blurriness and performance significantly, but the result may become too "washed out".</li>
<li>Set the blur radius by calling <code>setBlurRadius(x)</code>, where x is a <code>float</code> value between <code>1f</code> and <code>25f</code>.</li>
<li>Set additional options like an overlay color, drawable or resource, e.g. to also darken or light up the blurred view (for example by applying an alpha-added color like <code>#80000000</code>).</li>
</ul>
</li>
<li><p>Show the blurred View by calling <code>setOpacity(1f)</code>. You can enter any <code>float</code>value between 0f (hiding the view altogether) and 1f (for full opaqueness).</p></li>
</ol><hr><p>Code example:</p>

<pre><code>ViewBlurrer mViewBlurrer = new ViewBlurrer(getApplicationContext(), mRootLayout);
mViewBlurrer.setBlurScale(8);
mViewBlurrer.setBlurRadius(5f);
mViewBlurrer.setOverlayBackgroundColor(Color.parseColor("#99000000"));

mViewBlurrer.setOpacity(1f);
</code></pre>

<h1>
<a name="user-content-known-issues--limitations" class="anchor" href="#known-issues--limitations"><span class="octicon octicon-link"></span></a>Known issues / limitations</h1>

<ul>
<li>Creating a FrameLayout and an ImageView for every instance of ViewBlurrer can become somewhat costly, so try to keep the amount of blurred Views low (usually you would apply it to the root view of your activity; see examples).</li>
<li>Caution: Calling getParent() on the initial View will return the FrameLayout now, so call getParent() again to get the initial parent!</li>
</ul></article>
